Recipe Guide

GET /recipes

This route return a list of recipes that contain a recipeID, name, description, cooking time and the ID of the user that posted the recipe

GET /recipes/:id

This route returns a single recipe object.
It allows the user to see a certain recipe by entering its preticular ID in the path. This object also contains the name,description, cooking time and the ID of the user that adds the recipe

POST /recipes

This route allows the user to add a recipe to our database.
it utilizes the body of our request to create a new recipe object that contains everything our recipes should.
The body of our request should contain the name, description, and cooking time. When posted this object should add the ID of the user that has created the recipe.

PUT /recipes/:id

This route allows our user to edit the recipe they have added.
When this route is called you will be able to modify the values of the recipe with the specified ID. The user utilizes the body of the request to update the name, description, or cooking time.

DELETE /recipes/:id

This route allows you to delete the recipe with the specified ID
It will allow the user to delete a recipe from our database using the ID in the path paramater

/signup

This route allows the user to sign up and adds their username and password to our database.
DONT worry, once the password is passed into our database we hash the password that way in cases of a data breach we keep your information secure.
If the user leaves out their username or password it will prompt a message that the empty field is required.

/login

This route will allow the user to login to our application and provides them with a token that will verify the user. Once verified the user will have access to special actions of our application
If the user does not already exist in our database it will prompt a message that we could not find said user.

GET /categories

This route will allow the user to see the different categories our application holds
Each recipe will be applied a category tag that will then allow the user to see recipes that belong to a certain category.
Difficult recipes, Easy recipes, or the culture the recipe originates from

GET /ingredient

This route allows the user to see the list of recipes we already have. This will contain the name and a breif description of each ingredient.

POST /ingredient

This route allows the user to add an ingredient to our database.
It utilizes the body of the request to create and ingredient object that will contain the name and breif description. This ingredient will then be visible to other users when using the "GET /ingredient" route.
